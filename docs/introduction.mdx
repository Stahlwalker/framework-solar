Solar is a runtime UI framework built for AI-generated code. Typed contracts, structured errors, and a machine-readable component registry are built in.

Existing frameworks (React, Vue, Svelte) were built assuming a developer writes, reads, and maintains every file. AI-generated code works differently. It's often correct in isolation but inconsistent across files. It gets regenerated frequently instead of incrementally maintained. It doesn't respect conventions it can't see.

Solar makes contracts explicit, structure rigid, and side effects declared so agent output is predictably correct.

## Core principles

- **Explicit contracts:** props are typed and validated at the component boundary, not inferred
- **Declared side effects:** effects declare what they depend on and what they touch
- **Rigid file structure:** a model generating a new component knows exactly where everything goes
- **Runtime-based:** no compiler required; debuggable in the browser as-is
- **Small surface area:** fewer primitives means fewer ways to generate something wrong

The Button below has a typed schema. The first mount passes valid props. The second mount passes `label: 42`, a number where a string is required. Solar catches it before render and returns a structured `ContractError`. The `fix` field (green) is what an agent would read to self-correct.
